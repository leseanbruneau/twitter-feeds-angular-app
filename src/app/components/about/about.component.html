Angular

More information: Angular Framework

While certainly not required to set up a static web site, Angular framework help organize web site development. The more content added to the web site creates more complexity. Organizing a site into components helps break down the web site content into more manageable parts. This is where the Angular framework comes in.

Easiest way to get started is the Angular Quick Start guide. If you are interested in learning more about Angular, I would recommend a book or training video to get started building examples more details about Angular. I found a couple training videos on Udemy and Packt Publishing to be helpful.

Quick note regarding AWS S3 website hosting an Angular application. As mentioned earlier, AWS S3 website will not have root ("/") as default context path. By default, a new Angular application generated from ng-new will use a root context "/" path, which will not work with a simple AWS S3 website. To fix this situation, after creating a new Angular application, update the default index.html with your AWS S3 bucket name. Specifically, change:

base href="/" tag to base href="/%%YOUR_AWS_S3_BUCKET_NAME%%/"

This change will tell Angular application to add AWS S3 bucket name to the URL path that is appended on http://s3.amazonaws.com/ site.
