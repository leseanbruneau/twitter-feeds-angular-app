Quick Setup Using GitHub Repository Code

To use code from GitHub repository (run from command line, except for comment lines that start with #):

# Clone repo - will create a new twitter-feeds-angular-app directory in directory command is run from

git clone https://github.com/leseanbruneau/twitter-feeds-angular-app.git

# change directory to application root directory

cd twitter-feeds-angular-app

# install node_modules - will need Nodejs and Angular Cli installed on workstation

npm install --save-dev

# Run application on development server

ng serve

# Create distrobution build for deployment to web server - will create new dist directory in application root directory

ng build --prod

# Move all files in %%app-root-dir%%/dist/twitter-feeds-angular-app to web server

Web Hosting Note - AWS S3 website hosting

Quick note regarding AWS S3 website hosting an Angular application. As mentioned earlier, AWS S3 website will not have root ("/") as default context path. By default, a new Angular application generated from ng-new will use a root context "/" path, which will not work with a simple AWS S3 website. To fix this situation, after creating a new Angular application, update the default index.html with your AWS S3 bucket name. Specifically, change:

base href="/" tag to base href="/%%YOUR_AWS_S3_BUCKET_NAME%%/"

This change will tell Angular application to add AWS S3 bucket name to the URL path that is appended on http://s3.amazonaws.com/ site.
